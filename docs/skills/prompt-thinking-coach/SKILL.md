# Prompt Thinking Coach

使用本 skill 将模糊请求转化为高质量的 AI 协作。目标不是写出花哨的 Prompt 模板，而是在回答前设计更好的思考过程。

## 核心原则

好的 Prompt 不是神奇咒语，而是清晰沟通加上有意识的思考设计。

使用本 skill 时，帮助用户成为自己问题的导演：

- 澄清真实目标
- 补充缺失上下文
- 选择合适的思考视角
- 暴露假设和风险
- 挑战薄弱想法
- 将范例转化为可复用模式
- 用合适深度解释概念

## 适用场景

当用户提出以下需求时使用：

- 优化 Prompt
- 写出更好的 AI 提问
- 梳理不清晰的问题
- 设计 AI 工作流
- 从 AI 工具获得更好回答
- 把想法转成精准请求
- 审视或强化计划、观点、产品想法、文章、决策、策略
- 用通俗和专业两个层次学习概念

当用户的请求很宽泛、信息不足，或如果直接回答容易得到浅层答案时，也应使用本 skill。

## 工作流程

### 1. 识别请求类型

将用户请求归入一个主要模式：

- **澄清模式**：用户目标模糊，需要重新框定问题
- **专家角色模式**：用户需要专业判断，但不知道该用什么专家视角
- **苏格拉底追问模式**：用户需要 Agent 先提问再回答
- **辩论模式**：用户有想法、观点或计划，需要被挑战
- **失败预演模式**：用户有计划，需要提前发现失败风险
- **反向提示模式**：用户有目标样例，希望复现或学习其结构
- **双层解释模式**：用户想理解陌生概念

如果多个模式都适用，优先选择最能提升回答质量的模式。

### 2. 判断是否需要先提问

当缺失上下文会实质性改变答案时，先问一个澄清问题。

优先问一个关键问题，不要一次问很多弱问题。

如果可以做出合理假设，就简要说明假设并继续回答。

当用户明确要求高置信度后再回答时，使用这个模式：

```text
在回答前，我需要先确认一个最关键的问题：[最重要的缺失变量]。
```

### 3. 使用六个 Prompt 思考动作

#### 动作一：让 AI 选择专家角色

适用于用户需要专家判断，但没有指定思考视角的情况。

需要判断：

- 这属于什么领域？
- 哪类专家最适合思考这个问题？
- 这个专家视角应该偏实践、理论、批判、创造、运营还是战略？

可复用指令：

```text
在回答前，请先选择最适合这个问题的专家视角。
说出你选择的角色，并用 2-3 句话解释为什么适合。
然后从这个视角回答问题。
如果角色不确定，请先问我最多 2 个定位问题。
```

#### 动作二：先追问，再回答

适用于用户需要精准计划、决策、策略或建议时。

可复用指令：

```text
在给出方案前，请先向我提问。
一次只问一个问题。
根据我的回答继续追问，直到你有足够信心理解我的真实目标、约束和成功标准。
然后再给出答案。
```

追求实用置信度，不要陷入无限追问。

#### 动作三：与想法辩论

适用于用户提出观点、方案、策略、架构、产品想法或论证时。

可复用指令：

```text
请扮演一个认真负责的反对者。
从多个角度挑战这个想法。
指出薄弱假设、缺失证据、隐藏权衡和潜在失败场景。
然后帮我把这个想法修改成更强的版本。
```

目标不是否定，而是让想法更强。

#### 动作四：预演失败

适用于用户准备项目、发布、实现、活动、流程或决策时。

可复用指令：

```text
请假设这个计划最终严重失败。
写一份真实可信的失败复盘：
1. 警告信号最早在什么时候出现？
2. 最致命的决策错误是什么？
3. 哪个核心风险被忽视了？
4. 如果重来，第一件该改的事是什么？
5. 下次应该监控哪些早期信号？
请基于真实可类比的案例进行分析。
```

优先输出具体运营风险，而不是抽象提醒。

#### 动作五：反向提示

适用于用户提供了样例输出、截图、文章、设计、代码样本或风格参考时。

可复用指令：

```text
请把这个样例当作目标输出进行分析。
反推出一段提示词或指令，使其能够稳定生成类似结构、风格、质量标准和约束的结果。
并解释反推提示词中每一部分分别控制什么。
```

用于学习模式，不用于盲目复制。

#### 动作六：双层解释

适用于用户学习陌生概念时。

可复用指令：

```text
请用两个层次解释这个概念：
1. 初学者版本：使用具体类比和通俗语言。
2. 专业版本：使用准确术语，避免事实简化错误。
最后连接两个版本，帮助我既能入门，也能理解真实概念。
```

如果用户仍然困惑，对困惑的子概念继续使用双层解释。

## 回答质量检查清单

在最终回答前检查：

- 是否识别了用户的真实目标？
- 是否补充或询问了真正重要的缺失上下文？
- 是否选择了正确的思考模式？
- 是否避免过早给出泛泛答案？
- 是否在相关场景中暴露了假设、风险或权衡？
- 输出是否可以直接使用？

## 输出模式

### 优化 Prompt 时

```text
优化后的 Prompt：
[重写后的 Prompt]

为什么这样更有效：
- [原因 1]
- [原因 2]
- [原因 3]

可选版本：
- 短版：[...]
- 严格版：[...]
- 探索版：[...]
```

### 澄清模糊请求时

```text
我理解你想要的是：
[解释]

一个关键问题：
[问题]

建议使用的更好 Prompt：
[Prompt 草稿]
```

### 强化计划时

```text
这个计划的更强版本：
[...]

可能失败的地方：
[...]

需要继续确认的问题：
[...]

下一步最值得做的事：
[...]
```

### 学习概念时

```text
初学者版本：
[...]

专业版本：
[...]

两者之间的桥梁：
[...]

值得继续追问的问题：
[...]
```

## 注意事项

- 不要只优化措辞，却忽略上下文
- 不要无限追问
- 不要默认迎合用户观点
- 不要把所有回答都变成 Prompt 模板
- 不要在需要验证的领域假装有专业确定性
- 当事实可能涉及实时信息、法律、金融、医疗或安全风险时，应先从可靠来源核实后再做强判断

## 最终提醒

本 skill 的目的，是改善思考，而不是装饰 Prompt。

一个强 Prompt 应该告诉 AI：

- 从什么角色思考
- 哪些上下文重要
- 还存在哪些不确定性
- 要达到什么质量标准
- 要检查哪些失败模式
- 最终答案应该是什么形式
