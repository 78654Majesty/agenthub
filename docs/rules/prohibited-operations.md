# 全局禁止操作明细

## 概述

本文件定义了 AgentHub 项目中的全局禁止操作，所有开发人员必须严格遵守。

## 禁止操作列表

### 1. 禁止直接修改数据库 Schema

**操作**: 修改 `prisma/schema.prisma` 文件

**原因**: 
- 数据库 Schema 由项目负责人统一管理，确保数据结构的一致性和可维护性
- 未经审核的 Schema 变更可能导致数据丢失或系统不稳定

**正确做法**: 
- 在 PR 描述中列出需求（表名、字段名、类型、默认值）
- 项目负责人审核后修改 Schema 并合入 main
- 各分支 rebase 获取最新 Schema

### 2. 禁止修改 Gateway 入口文件

**操作**: 修改 `gateway/src/index.ts` 文件

**原因**: 
- 入口文件由 route-loader 自动加载路由，手动修改可能破坏路由加载机制
- 统一的路由加载机制确保了代码的一致性和可维护性

**正确做法**: 
- 遵循现有的路由加载机制
- 在指定的路由目录下添加新路由

### 3. 禁止跨模块修改

**操作**: 修改不属于自己负责模块的代码

**原因**: 
- 确保代码责任清晰，便于问题追踪和维护
- 避免多个人同时修改同一模块导致的冲突
- 保护各模块的稳定性和完整性

**正确做法**: 
- 只在自己独占的目录下编写代码
- 如需跨模块协作，先与相关负责人沟通

### 4. 禁止直接操作链上资源

**操作**: 绕过 `packages/chain/` 封装直接进行 ERC-8004 / Solana 操作

**原因**: 
- 链上操作封装确保了安全性和一致性
- 统一的接口可以更好地处理错误和异常情况
- 便于后续升级和维护

**正确做法**: 
- 所有链上操作必须通过 `packages/chain/` 提供的接口进行
- 如需新增链上功能，先扩展 `packages/chain/` 封装

### 5. 禁止在共享文件中添加模块特定逻辑

**操作**: 在 `packages/auth`、`packages/chain` 等共享包中添加特定模块的逻辑

**原因**: 
- 共享包应该保持通用性，不依赖于特定模块
- 模块特定逻辑会增加共享包的复杂度和耦合度
- 降低了共享包的可复用性

**正确做法**: 
- 共享包只提供通用功能和接口
- 模块特定逻辑应该放在各自的模块目录下
- 通过组合方式使用共享包功能

## 违反后果

- 可能导致代码冲突和合并困难
- 影响系统的稳定性和可靠性
- 增加维护成本和技术债务
- 严重违反者可能影响项目进度和团队协作

## 举报机制

如发现有人违反上述禁止操作，请及时与项目负责人联系。
